# YAUZ i18n Guidelines

This project supports three languages. All user-facing text in `dist/index.html` must go through the `I18N` system.

## Supported Locales

| Code    | Language            |
|---------|---------------------|
| zh-CN   | Simplified Chinese  |
| zh-TW   | Traditional Chinese |
| en-GB   | British English     |

## Rules

1. Never hardcode user-facing strings directly in HTML or JS. Always add a key to the `I18N` object in `dist/index.html`.
2. For every new text key, provide translations for all three locales: `zh-CN`, `zh-TW`, and `en-GB`.
3. Static text in HTML uses `data-i18n="key_name"` attributes, applied by `applyI18n()`.
4. Dynamic text in JS uses `t.key_name` (string) or `t.key_name(arg)` (function) from the current locale.
5. Placeholders for `<input>` and `<textarea>` are set in `applyI18n()` via JS, not HTML attributes.
6. Language detection is automatic via `navigator.languages`, mapping to the closest supported locale.
7. Use British English spelling (e.g. "colour", "organisation", "recognise") for `en-GB` translations.
8. Use Traditional Chinese characters and Taiwanese/HK terminology for `zh-TW` (e.g. 檔案 not 文件, 資料夾 not 文件夹).
